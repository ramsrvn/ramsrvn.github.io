Sharwan Ram is a highly skilled Software Engineer 2 in Product Security at [Garmin International](https://www.garmin.com/), specializing in Static Analysis (SAST), Software Composition Analysis (SCA), Software Bill of Materials (SBOM), automation, and vulnerability management. He led the enterprise-wide integration of Cppcheck, improving code quality and security, and developed automation scripts and a Python package to streamline static analysis workflows and optimize issue prioritization.
Additionally, he mentored team member on product security processes and tools, fostering a culture of security best practices and continuous improvement.

Sharwan is driving SBOM integration and adoption, developing automation scripts to streamline its integration and enhance third-party risk management. He also maintains critical security libraries used across enterprise products, ensuring compliance and proactive vulnerability prevention. He actively participates in EU Radio Equipment Directive (RED), Cyber Resilience Act (CRA), US Cyber Trust Mark, and Executive Order 14028 compliance discussions.

Previously, Sharwan worked as a Software Engineer at [EnsoftCorp](http://ensoftcorp.com/), where he optimized legacy systems and developed Eclipse plug-ins for software analysis. He has also served as a Software Developer and Security Analyst on the [DARPA's Space/Time Analysis for Cybersecurity (STAC)](http://www.darpa.mil/program/space-time-analysis-for-cybersecurity) program at Iowa State University, developing advanced program analyzers to detect sophisticated vulnerabilities, he was a top-performing team member.

Sharwan holds two master’s degrees: an M.S. from Iowa State University and an M.Tech from MNIT Jaipur. He received recognition from [MIT](https://www.mit.edu/) for his contributions to the [Kerberos](https://github.com/krb5/krb5). His research on automated documentation generation earned the IEEE Best Challenge Entry Award at DySDoc3.
